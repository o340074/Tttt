# 00 — Обзор проекта

## Резюме

**AdsMarket** — веб-платформа для продажи цифровых товаров, стилизованная под
интерфейс Google Ads. Пользователь заходит на сайт, видит каталог рекламных
аккаунтов и других цифровых товаров, пополняет внутренний баланс (или платит
криптой напрямую), оформляет заказ и получает товар: либо мгновенно из стока,
либо после ручной выдачи продавцом.

Платформа должна выглядеть **профессионально и доверенно** — как настоящий
B2B-продукт Google, а не как кустарный «автомагазин».

## Что это НЕ (границы этапа)

Эта сессия — **только планирование**. Мы не пишем прод-код, не поднимаем
инфраструктуру, не подключаем реальные платёжные шлюзы. Мы создаём документацию,
по которой следующие сессии смогут быстро и предсказуемо собрать продукт.

## Глоссарий

| Термин | Значение |
|--------|----------|
| **Товар (Product)** | Позиция в каталоге. Может иметь несколько вариантов (SKU). |
| **Вариант / SKU** | Конкретная разновидность товара с ценой и типом выдачи. |
| **Сток (Stock item)** | Единица товара для авто-выдачи (например, строка `email:pass`). |
| **Заказ (Order)** | Факт покупки: что, сколько, кем, на какую сумму, статус. |
| **Выдача (Delivery)** | Передача купленного товара покупателю (авто или вручную). |
| **Баланс / Кошелёк** | Внутренние средства пользователя, пополняемые криптой. |
| **Пополнение (Top-up)** | Транзакция зачисления средств на баланс. |
| **Тикет (Ticket)** | Обращение в поддержку. |
| **Категория** | Раздел каталога (напр. «Google Ads», «Facebook», «Прокси»). |
| **Реселлер / Продавец** | Роль, если платформа мультивендорная (см. v2/v3). |

## Роли пользователей

- **Гость** — просматривает каталог, регистрируется.
- **Покупатель (User)** — покупает, пополняет баланс, открывает тикеты.
- **Поддержка (Support)** — отвечает на тикеты, обрабатывает ручные выдачи.
- **Администратор (Admin)** — управляет товарами, стоком, ценами, пользователями,
  видит финансы и статистику.
- **(v2) Продавец (Vendor)** — если делаем мультивендорный маркетплейс.

## Как пользоваться этой документацией

1. Читай по порядку от 00 к 10.
2. Документы 01–02 отвечают на «что и зачем».
3. Документы 03–07 отвечают на «как технически».
4. Документы 08–09 — про деньги, доставку, риски.
5. Документ 10 — операционный план: с чего начинать следующую сессию.

Каждый документ самодостаточен, но ссылается на соседние. Если что-то меняется —
правь соответствующий документ и отмечай изменение, чтобы будущие сессии видели
актуальное состояние.
