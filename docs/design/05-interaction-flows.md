# Design 05 — Сценарии взаимодействия: каждое действие

Здесь расписано поведение **каждой значимой кнопки/действия**: триггер → что
происходит → состояния (loading/success/error) → анимация → edge-cases. Формат
компактный, но исчерпывающий.

## Обозначения
- **T** — триггер (что нажал пользователь)
- **→** — результат/переходы состояний
- **⚠** — edge-cases и ошибки
- **✦** — анимация/фидбек

---

## Аутентификация

### Регистрация
- **T:** кнопка «Sign up» на форме.
- → валидация полей (email формат, пароль ≥ 8, совпадение) → кнопка loading →
  запрос → success: экран «проверьте email» + toast.
- ⚠ email занят → ошибка под полем + shake; слабый пароль → индикатор силы;
  сеть упала → toast retry.
- ✦ ошибки — shake полей + Coral-glow; успех — success-flash.

### Вход
- **T:** «Log in». → валидация → loading → success: редирект в ЛК + toast «Welcome».
- ⚠ неверные данные → общая ошибка (без раскрытия, что именно) + shake; много
  попыток → rate-limit сообщение + таймер; не подтверждён email → предложить
  переотправку письма.

### Восстановление пароля
- **T:** «Forgot password» → форма email → «Send link» → toast «Если email есть,
  письмо отправлено» (без раскрытия существования). Экран сброса по токену:
  новый пароль + подтверждение → success → редирект на вход.

### Выход
- **T:** «Log out» в меню профиля → confirm (опц.) → очистка токенов → редирект на
  главную + toast.

## Каталог и товар

### Поиск
- **T:** ввод в строку поиска. → debounce 300ms → live-подсказки/результаты →
  очистка по крестику. ⚠ нет результатов → EmptyState. ✦ подсветка совпадений.

### Применение фильтра
- **T:** чек категории / сдвиг range-цены / выбор гео. → URL-параметры обновляются →
  сетка перезапрашивается → chips активных фильтров. «Сбросить всё» очищает.
- ✦ cross-fade сетки, skeleton при загрузке. ⚠ пустой результат → EmptyState + сброс.

### Сортировка
- **T:** select «Сортировать». → пересортировка (цена ↑/↓, новизна, рейтинг) →
  плавное перестроение сетки.

### Открыть товар
- **T:** клик по карточке. → переход на страницу товара (page-transition), sticky-блок
  покупки. ✦ shared-element (v2).

### Выбор варианта (SKU)
- **T:** переключение типа/гео/лимита. → пересчёт цены (count-up), обновление наличия
  и типа выдачи (auto/manual), доступность CTA. ⚠ вариант нет в наличии → CTA
  «Под заказ» или «Уведомить».

### Добавить в корзину
- **T:** «В корзину». → проверка → добавление → бейдж корзины +1 (пружина). ✦ «полёт»
  товара к иконке. ⚠ уже в корзине → увеличить кол-во; нет в наличии → блок.

### Купить сейчас
- **T:** «Купить сейчас». → если гость → предложить вход/регистрацию (с сохранением
  намерения) → иначе прямой checkout данной позиции.

## Корзина и оформление

### Изменить количество / удалить
- **T:** +/- или удалить. → пересчёт итога (count-up). ⚠ превышение доступного стока →
  ограничение + подсказка. ✦ удаление slide-out.

### Промокод
- **T:** ввод + «Применить». → проверка → success: скидка в итоге + бейдж кода;
  ошибка: «недействителен/истёк/лимит» под полем. ✦ пересчёт count-up.

### Checkout — выбор способа оплаты
- **T:** выбор «Баланс» или «Крипта». → показ соответствующего блока.
  - **Баланс:** показать баланс vs total. Если хватает → «Оплатить». Если нет →
    «Пополнить на X» (ведёт в пополнение, сохраняя заказ).
  - **Крипта (P1):** генерация платежа, QR, ожидание подтверждения.

### Подтверждение оплаты с баланса
- **T:** «Оплатить». → loading (кнопка) → сервер: резерв стока + списание (транзакция) →
  success: **success-flash** экрана + переход к выдаче + toast + баланс count-up вниз.
- ⚠ недостаточно баланса → ошибка + CTA пополнить; товар разобрали (сток кончился в
  момент оплаты) → сообщение + перевод в ручную выдачу/возврат; двойной клик →
  идемпотентность (Idempotency-Key), второй запрос игнорируется.

## Выдача

### Авто-выдача (после оплаты)
- → данные появляются в «Купленных товарах»/деталях заказа сразу, статус delivered.
  ✦ success-flash, toast «Ваш товар готов».

### Ручная выдача (ожидание)
- → статус awaiting_manual, индикатор ожидания + ориентир по времени; уведомление
  при готовности (email + in-app + пульс-бейдж).

### Показать / Копировать / Скачать данные
- **T:** «Показать» → снятие маски (по клику, логируется). «Копировать» → в буфер +
  иконка→галочка + micro-flash + toast «Copied». «Скачать» → файл (txt).
- ⚠ доступ только владельцу; попытка чужого доступа → 403.

### Запросить замену (гарантия)
- **T:** «Заменить». → форма причины → отправка → статус «на проверке» → выдача новой
  единицы (replacement) или возврат на баланс. ⚠ вне гарантийного окна → блок с пояснением.

## Кошелёк и пополнение

### Создать пополнение
- **T:** сумма + актив/сеть + «Пополнить». → создание платежа → экран с QR/адресом +
  таймер + статус pending. ⚠ сумма < минимума → подсказка; провайдер недоступен →
  ошибка + retry.

### Ожидание подтверждения
- → поллинг/websocket статуса. При подтверждении: **Aurora-flash + confetti (опц.)** +
  баланс count-up + toast + смена статуса на confirmed. ⚠ истёк таймер → expired +
  «создать заново»; недоплата/переплата → обработка по политике провайдера + сообщение.

## Поддержка

### Создать тикет
- **T:** «Новый тикет» → тема + сообщение + (опц.) привязка заказа + вложения →
  «Отправить» → тикет open + переход в переписку.
### Ответить / вложение
- **T:** ввод + «Отправить». → сообщение добавляется (optimistic) → подтверждение;
  вложение — загрузка с прогрессом. ⚠ большой файл → ошибка размера.
### Закрыть/переоткрыть тикет
- статусы меняются, уведомления сторонам.

## Профиль и настройки

- **Смена пароля:** старый+новый → валидация → success/ошибка.
- **Смена языка:** мгновенное переключение интерфейса (i18n), сохранение в профиль.
- **Смена темы:** dark/light toggle, плавный переход цветов.
- **2FA (v2):** включение через QR TOTP + подтверждение кодом.
- **Уведомления:** переключатели каналов.

## Админ (ключевые действия)

- **CRUD товара/категории:** формы с валидацией, переводы EN/RU, публикация/черновик,
  подтверждение удаления (danger-confirm).
- **Импорт стоков:** drag-n-drop CSV/TXT → парсинг → отчёт (успешно N, ошибки в
  строках M) → подтверждение импорта.
- **Ручная выдача:** открыть заказ awaiting_manual → ввести payload → «Выдать» →
  статус delivered + уведомление покупателю. ⚠ повторная выдача блокируется.
- **Возврат:** «Вернуть» → сумма + причина → ledger-credit + статус refunded + аудит.
- **Корректировка баланса:** сумма + причина (обязательна) → ledger + audit-log.
- **Блокировка пользователя:** confirm → статус blocked, сессии отзываются.

## Глобальные действия/состояния

- **Навигация между страницами:** top progress bar + cross-fade.
- **Смена языка/темы:** без перезагрузки, плавно.
- **Потеря сети:** баннер offline + очередь повторов для безопасных запросов.
- **Сессия истекла:** тихий refresh; если не вышло → модалка «войдите снова».
- **Ошибка сервера (5xx):** toast + возможность повторить; критично — экран ошибки.
- **Пустые состояния:** у каждого списка (заказы/корзина/тикеты/поиск) — свой EmptyState с CTA.
- **Подтверждение необратимого:** всегда danger-confirm (удаление, блокировка, возврат).
