# Design 06 — Доступность и адаптивность

## Доступность (a11y) — цель WCAG 2.1 AA

### Контраст и цвет
- Текст к фону: ≥ 4.5:1 (обычный), ≥ 3:1 (крупный). Тёмная тема проверена под это.
- Не полагаться только на цвет: статусы дублируются иконкой/текстом (paid+галочка).
- Glow/flash — декоративны, не несут единственного смысла.

### Клавиатура
- Все интерактивные элементы достижимы Tab, логичный порядок.
- Видимый фокус везде (кольцо Volt + glow), никогда не `outline:none` без замены.
- Модалки — focus-trap + возврат фокуса; Escape закрывает; меню — стрелки.
- Пропуск-ссылка «Skip to content» в начале.

### Семантика и ARIA
- Нативные элементы (`button`, `a`, `input`, `table`) прежде ARIA.
- Роли для кастомных: tablist/tab/tabpanel, listbox/option, dialog, status.
- Формы: `label`+`for`, `aria-invalid`, `aria-describedby` для ошибок/подсказок.
- Живые области: toast `aria-live=polite` (ошибки `assertive`), загрузки `aria-busy`.
- Иконки-кнопки имеют `aria-label`.

### Контент
- Альт-тексты для смысловых изображений; декоративные — `alt=""`.
- Понятные тексты ссылок/кнопок (не «сюда»).
- Язык страницы в `lang`, переключение обновляет `lang`.

### Движение
- `prefers-reduced-motion: reduce` → отключение параллакса/particles/крупных
  перемещений; мгновенные fade без сдвигов. Функциональность сохраняется.

### Формы и ошибки
- Ошибки — текстом рядом с полем + фокус на первую ошибку + суммарное сообщение.
- Таймауты/лимиты объявляются заранее, с возможностью продлить где уместно.

### Тестирование a11y
- Автоматически: axe-core в CI/e2e.
- Ручное: клавиатура-онли проход ключевых сценариев, проверка скринридером
  (VoiceOver/NVDA) для auth, checkout, выдачи.

## Адаптивность (responsive)

### Брейкпоинты (из токенов)
`xs(0) · sm(640) · md(768) · lg(1024) · xl(1280) · 2xl(1536)`

### Принципы
- **Mobile-first:** базовые стили — под мобилку, наращиваем на больших экранах.
- Контент не «прыгает»: резерв места под изображения/скелетоны (без CLS).
- Горизонтального скролла у body быть не должно; широкое (таблицы, диаграммы) —
  в собственном `overflow-x:auto` контейнере.

### Поведение по зонам
| Зона | Mobile | Tablet | Desktop |
|------|--------|--------|---------|
| App bar | бургер + иконки | + поиск | полный (поиск по центру) |
| Каталог сетка | 1 кол. | 2 кол. | 3–4 кол. |
| Фильтры | drawer по кнопке | drawer/панель | боковая панель |
| Товар | 1 колонка, sticky CTA внизу | 2 колонки | 2 колонки, sticky блок |
| Checkout | степпер вертикально, сводка снизу | 2 колонки | 2 колонки |
| ЛК/Админ сайдбар | bottom nav / drawer | свернутый | полный |
| Таблицы | скролл + «липкий» 1-й столбец | скролл | полная |

### Тач-адаптация
- Тач-таргеты ≥ 44×44px, увеличенные отступы кликабельных зон.
- Hover-эффекты имеют тач-эквивалент (active), не зависящий от наведения.
- Свайпы: закрытие drawer/модалок, навигация галереи товара.

### Плотность
- Комфортная плотность по умолчанию; в админ-таблицах — опция «компактно».

### Изображения/медиа
- `max-width:100%`, адаптивные размеры, ленивая загрузка ниже сгиба.
- Иконки — векторные (SVG), чёткие на любом DPI.

## Производительность (влияет на UX)
- Ленивая подгрузка тяжёлых экранов/админки (code-splitting).
- Кэш каталога (React Query) + скелетоны вместо пустых экранов.
- Ограничение тяжёлых анимаций на слабых устройствах (по `deviceMemory`/FPS-гварду).
- Целевые метрики: LCP < 2.5s, CLS < 0.1, INP < 200ms.
