# Design 04 — Каталог экранов

Каждый экран: **цель → лейаут → элементы → состояния → отзывчивость → анимации входа**.
Отсылки к компонентам — из [03](./03-component-library.md), к действиям — из
[05](./05-interaction-flows.md).

## Публичная зона

### S1. Лендинг / Главная
- **Цель:** первое впечатление, доверие, вход в каталог.
- **Лейаут (сверху вниз):**
  1. Top App Bar (прозрачный поверх hero).
  2. **Hero:** крупный заголовок (Aurora-градиентный акцент), подзаголовок, CTA
     «Browse catalog» + «Sign up», анимированный Aurora-фон, floating-карточки
     товаров с parallax.
  3. **Trust-бар:** цифры (аккаунтов продано, аптайм, средняя выдача), логотипы
     способов оплаты.
  4. **Категории:** плитки с иконками и glow-hover.
  5. **Популярные товары:** сетка ProductCard (stagger).
  6. **Как это работает:** 3–4 шага (выбор → оплата → мгновенная выдача → гарантия).
  7. **Преимущества:** мгновенная выдача, крипта, поддержка 24/7, гарантия замены.
  8. **Отзывы/соц.доказательство.**
  9. **FAQ (accordion).**
  10. **Футер:** навигация, языки, правовые ссылки.
- **Состояния:** статичный контент + подгрузка популярных (skeleton).
- **Отзывчивость:** hero-текст уменьшается, floating-карточки скрываются на mobile.
- **Анимации:** hero fade-up + aurora-shift; секции появляются on-scroll.

### S2. Каталог (список товаров)
- **Цель:** найти и отфильтровать товар.
- **Лейаут:** слева панель фильтров, справа сетка ProductCard; сверху — строка
  поиска, сортировка, счётчик результатов, активные фильтры-чипы.
- **Фильтры:** категория, цена (range-slider), гео, тип аккаунта, наличие, рейтинг.
- **Состояния:** loading (skeleton-сетка), empty («ничего не найдено» + сброс),
  error (retry), подгрузка след. страницы.
- **Отзывчивость:** фильтры → drawer по кнопке на mobile; сетка 1→2→3→4 колонки.
- **Анимации:** stagger fade-up карточек; смена фильтра — cross-fade сетки.

### S3. Страница товара
- **Цель:** решение о покупке.
- **Лейаут:** слева галерея/иконка + бейджи; справа — заголовок, рейтинг, выбор
  варианта (SKU: тип/гео/лимит), цена, наличие, кнопки «Купить сейчас» и «В корзину»,
  блок гарантий/выдачи. Ниже — описание, характеристики (таблица), отзывы, похожие.
- **Состояния:** в наличии / под заказ / нет в наличии (CTA меняется), loading.
- **Отзывчивость:** двухколоночный → одноколоночный; sticky-блок покупки.
- **Анимации:** shared-element из карточки (v2), смена SKU — плавное обновление цены (count-up).

### S4. Корзина
- **Цель:** проверить и оформить.
- **Лейаут:** список позиций (кол-во, цена, удаление), итог справа (subtotal, скидка,
  total), промокод, кнопка «Оформить».
- **Состояния:** пустая (EmptyState + «В каталог»), пересчёт при изменении, товар
  стал недоступен (пометка + предложение убрать).
- **Анимации:** удаление — slide-out + пересчёт count-up.

### S5. Оформление / Checkout
- **Цель:** оплатить.
- **Лейаут:** stepper (Обзор → Оплата → Готово); выбор способа (Баланс / Крипта);
  сводка заказа; при «Баланс» — текущий баланс и «оплатить»; при нехватке — CTA
  «Пополнить». Кнопка подтверждения.
- **Состояния:** проверка наличия, обработка оплаты (loading), успех (success-flash +
  переход к выдаче), ошибка (shake + toast).
- **Отзывчивость:** сводка уходит вниз на mobile.

## Личный кабинет (авторизован)

### S6. Дашборд ЛК
- **Цель:** обзор аккаунта.
- **Лейаут:** сайдбар + контент. Карточки: Баланс (+ «Пополнить»), последние заказы,
  быстрые действия, статус тикетов.
- **Анимации:** count-up баланса, появление карточек stagger.

### S7. Мои заказы
- **Лейаут:** таблица/список заказов (номер, дата, сумма, статус, выдача), фильтры,
  переход в детали.
- **Состояния:** empty, loading, статусы бейджами.

### S8. Детали заказа
- **Лейаут:** позиции, статус выдачи каждой, кнопки «Показать данные»/«Копировать»/
  «Скачать», «Запросить замену», связанные тикеты.
- **Состояния:** delivered / awaiting_manual (индикатор ожидания) / replaced.

### S9. Купленные товары (Vault)
- **Цель:** доступ к купленным данным.
- **Лейаут:** список выданных единиц, secret-field (маска + показать/копировать/
  скачать), фильтр по товару/дате.
- **Анимации:** micro-flash при копировании.

### S10. Кошелёк / Баланс
- **Лейаут:** крупный баланс, «Пополнить», история транзакций (ledger: пополнения/
  списания/возвраты) таблицей, фильтры.
- **Анимации:** count-up, подсветка новой транзакции.

### S11. Пополнение баланса
- **Цель:** внести крипту.
- **Лейаут:** сумма, выбор актива/сети, генерация адреса → QR + адрес + таймер +
  статус (pending/confirmed), инструкция.
- **Состояния:** created → waiting → confirmed (Aurora-flash + count-up) / expired.

### S12. Тикеты поддержки
- **Лейаут:** список тикетов (статус, тема, обновление) + окно переписки (чат-стиль),
  создание нового, вложения, привязка к заказу.
- **Состояния:** open/pending/closed, новое сообщение (подсветка).

### S13. Профиль и настройки
- **Лейаут:** данные, смена пароля, 2FA (v2), язык, тема, уведомления.

### S14. Auth-экраны
- Регистрация, вход, подтверждение email, восстановление пароля. Центрированная
  карточка на Aurora-фоне, плавающие лейблы, валидация, состояния загрузки/ошибок.

## Админ-панель

### A1. Дашборд
- KPI-карточки (выручка, заказы, конверсия, ожидают выдачи), графики, последние
  события, очередь ручных выдач и тикетов.

### A2. Товары и категории
- Таблица товаров (CRUD), редактор товара (переводы EN/RU, атрибуты, варианты/SKU,
  тип выдачи, цена), управление категориями (дерево).

### A3. Сток
- Список единиц по вариантам (available/reserved/sold), импорт CSV/TXT (drag-n-drop
  + отчёт по строкам), просмотр (payload — под доступом/аудитом).

### A4. Заказы
- Таблица заказов, фильтры, детали, **ручная выдача** (ввод payload), отмена, возврат.

### A5. Очередь ручных выдач
- Приоритетная очередь awaiting_manual, быстрый ввод данных, SLA-таймеры.

### A6. Пользователи
- Список, поиск, блокировка, корректировка баланса (с причиной → ledger/audit).

### A7. Финансы
- Пополнения, списания, возвраты, сходимость (ledger), экспорт.

### A8. Промокоды (v2), A9. Тикеты (очередь), A10. Аудит-лог, A11. Настройки/переводы.

## Служебные экраны
- 404, 403, 500, обслуживание (maintenance), offline. Все — в брендовом стиле с
  иллюстрацией и действием возврата.
