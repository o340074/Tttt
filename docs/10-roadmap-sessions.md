# 10 — Дорожная карта по сессиям

Операционный план: как разбить разработку на последующие сессии так, чтобы каждая
была самодостаточной, начиналась не «с нуля» и оставляла проект в рабочем состоянии.

> **Правило для каждой сессии:** в начале — прочитать этот файл и релевантные доки;
> в конце — обновить статусы ниже, закоммитить и запушить в рабочую ветку.

## Легенда статусов
- ⬜ не начато · 🟡 в работе · ✅ готово

## Фаза 0 — Планирование (текущая сессия)
- ✅ Видение, границы, фичи, стек, архитектура, модель данных, дизайн, API, оплата,
  безопасность, дорожная карта задокументированы.

## Фаза 1 — Каркас проекта (Сессия 1)
Цель: поднять монорепо и «hello world» с обеих сторон.
- ⬜ Инициализировать pnpm-монорепо (`apps/web`, `apps/api`, `packages/*`).
- ⬜ Настроить ESLint/Prettier/tsconfig, общие конфиги.
- ⬜ `apps/api`: NestJS-скелет, health-check, подключение Postgres (Prisma) и Redis.
- ⬜ `apps/web`: Vite + React + Tailwind + роутинг + i18n (EN/RU), базовый лейаут.
- ⬜ Docker Compose (postgres, redis, api, web) + локальный запуск.
- ⬜ CI (GitHub Actions): линт + сборка.
- **Выход:** локально поднимается, фронт видит `/health` бэка.

## Фаза 2 — Аутентификация и пользователи (Сессия 2)
- ⬜ Prisma-схема: User, роли, статусы.
- ⬜ Регистрация, вход, JWT (access+refresh), refresh-ротация.
- ⬜ Подтверждение email, восстановление пароля (email-заглушка/провайдер).
- ⬜ Фронт: страницы регистрации/входа, хранение токенов, guard-роуты.
- ⬜ Профиль `/me`, выбор локали.
- **Выход:** можно зарегистрироваться, войти, увидеть пустой ЛК.

## Фаза 3 — Каталог (Сессия 3)
- ⬜ Prisma: Category, Product, ProductVariant, *Translation.
- ⬜ API каталога: категории, список/карточка товара, фильтры, поиск.
- ⬜ Фронт: витрина, каталог с фильтрами, карточка товара, i18n-контент.
- ⬜ Сидер демо-данных.
- **Выход:** каталог просматривается, работают фильтры и локализация.

## Фаза 4 — Кошелёк и пополнение криптой (Сессия 4)
- ⬜ Prisma: LedgerEntry, TopUp.
- ⬜ Выбор и интеграция крипто-эквайринга (sandbox).
- ⬜ Создание пополнения, вебхук с проверкой подписи, идемпотентность.
- ⬜ Ledger + пересчёт баланса в транзакции БД.
- ⬜ Фронт: экран пополнения (QR/адрес, статус), история транзакций.
- **Выход:** тестовое пополнение доходит до баланса корректно.

## Фаза 5 — Корзина, заказы, оплата с баланса (Сессия 5)
- ⬜ Prisma: Order, OrderItem.
- ⬜ Корзина, checkout: проверка наличия, списание с баланса (ledger + транзакция).
- ⬜ Фронт: корзина, оформление, история заказов.
- **Выход:** можно купить товар за баланс, заказ создаётся и оплачивается.

## Фаза 6 — Выдача товара (Сессия 6)
- ⬜ Prisma: StockItem, Delivery + шифрование payload.
- ⬜ Резерв стока (Redis TTL) и авто-выдача после оплаты.
- ⬜ Ручная выдача: очередь задач + выдача из админки.
- ⬜ Фронт: «Купленные товары» (показ/копирование/скачивание), статусы выдачи.
- ⬜ Импорт стоков (CSV/TXT) в админке.
- **Выход:** авто- и ручная выдача работают end-to-end.

## Фаза 7 — Админ-панель (Сессия 7)
- ⬜ RBAC-гварды, лейаут админки.
- ⬜ CRUD категорий/товаров/вариантов, управление стоком.
- ⬜ Управление заказами (выдача, отмена, возврат), пользователями, финансами.
- ⬜ Базовый дашборд статистики, аудит-лог.
- **Выход:** магазином можно управлять без прямого доступа к БД.

## Фаза 8 — Поддержка и уведомления (Сессия 8)
- ⬜ Prisma: Ticket, TicketMessage.
- ⬜ Тикеты (создание, переписка, привязка к заказу) + очередь в админке.
- ⬜ Транзакционные email (регистрация, оплата, выдача, тикеты).
- **Выход:** пользователь может обратиться в поддержку, приходят письма.

## Фаза 9 — Полировка, безопасность, запуск (Сессия 9)
- ⬜ Отзывы/рейтинг (базово), промокоды (при необходимости).
- ⬜ Rate limiting, заголовки безопасности, security-review (`/security-review`).
- ⬜ E2E-тесты ключевых потоков (Playwright), нагрузочная проверка выдачи/оплаты.
- ⬜ Юридические страницы (ToS/Privacy/Refund), тёмная тема (опц.).
- ⬜ Продакшн-деплой, мониторинг, бэкапы.
- **Выход:** MVP готов к запуску.

## Дальнейшее (v2/v3, после MVP)
- Прямая оплата криптой за заказ, оплата картой (после оценки рисков).
- Реферальная программа, расширенная аналитика.
- Мультивендорность, выплаты продавцам.
- Мобильное приложение поверх API.
- Автопроверка/прогрев аккаунтов, гарантийная автозамена.

## Оценка масштаба
- MVP ≈ Фазы 1–9. Каждая фаза — примерно одна фокус-сессия (может делиться на под-сессии).
- Порядок фаз можно слегка тасовать, но зависимости: Auth → Каталог → Кошелёк →
  Заказы → Выдача → Админка. Поддержку/уведомления можно частично вести параллельно.
