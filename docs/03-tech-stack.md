# 03 — Технологический стек

Выбран вариант **«React (фронт) + отдельный бэкенд»**. Ниже — конкретные технологии
и обоснование. Один язык (TypeScript) на всём стеке снижает порог входа и упрощает
переиспользование типов.

## Итоговый стек (рекомендация)

### Фронтенд
- **React 18 + TypeScript** — стандарт индустрии, огромная экосистема.
- **Vite** — быстрый билд и dev-сервер.
- **React Router** — маршрутизация SPA.
- **TanStack Query (React Query)** — работа с серверным состоянием, кэш, ретраи.
- **Zustand** — лёгкое клиентское состояние (корзина, UI).
- **Tailwind CSS** — утилитарные стили, быстро повторить Google-эстетику.
- **shadcn/ui + Radix** — доступные примитивы компонентов, кастомизируются под дизайн.
- **react-i18next** — мультиязычность (EN по умолчанию, RU и др.).
- **React Hook Form + Zod** — формы и валидация (Zod общий с бэком).

### Бэкенд
- **Node.js + NestJS (TypeScript)** — структурированный фреймворк: модули, DI,
  guards, pipes. Хорошо масштабируется по мере роста фич.
- **PostgreSQL** — надёжная реляционная БД, отлично подходит под заказы/финансы.
- **Prisma ORM** — типобезопасный доступ к БД, миграции, удобная схема.
- **Redis** — кэш, сессии/refresh-токены, очереди, rate limiting, резервы стока.
- **BullMQ** — фоновые задачи (email, вебхуки, обработка выдач).
- **Zod / class-validator** — валидация входных данных.
- **JWT (access + refresh)** — аутентификация; refresh хранится в Redis/HTTP-only cookie.

### Инфраструктура и инструменты
- **Docker + Docker Compose** — одинаковое окружение dev/prod.
- **Nginx** — reverse proxy, TLS, статика фронта.
- **GitHub Actions** — CI: линт, тесты, сборка.
- **Sentry** — трекинг ошибок фронта и бэка.
- **Playwright** — e2e-тесты; **Vitest/Jest** — юнит-тесты.
- **ESLint + Prettier** — единый стиль кода.

### Платежи
- **Крипто-эквайринг** — например, Cryptomus / NOWPayments / Coinbase Commerce.
  Выбор конкретного провайдера — в [08](./08-payments-delivery.md). Интеграция
  через API + вебхуки подтверждения.

## Почему именно так

| Решение | Причина |
|---------|---------|
| TS на фронте и бэке | Общие типы/схемы, меньше рассинхрона, один язык. |
| NestJS | Готовая модульная структура под растущий проект, DI, тестируемость. |
| PostgreSQL | Транзакции, целостность для финансов и заказов. |
| Prisma | Быстрые миграции, типобезопасность, читаемая схема. |
| Redis + BullMQ | Резерв стока, очереди вебхуков/писем, антифрод-лимиты. |
| Tailwind + shadcn | Быстро собрать чистый Google-подобный UI, легко темизировать. |
| react-i18next | Зрелое решение i18n с ленивой загрузкой языков. |

## Структура репозитория (монорепо)

```
/
├── apps/
│   ├── web/          # React + Vite (фронтенд)
│   └── api/          # NestJS (бэкенд)
├── packages/
│   ├── types/        # Общие TS-типы / Zod-схемы
│   └── config/       # ESLint, tsconfig, общие конфиги
├── docs/             # Эта документация
├── docker-compose.yml
└── package.json      # workspaces (pnpm)
```

**Менеджер пакетов:** pnpm (workspaces) — быстрый, экономит место, удобен для монорепо.

## Альтернативы, которые рассмотрели и отклонили

- **Next.js фуллстек** — отклонён в пользу явного разделения фронт/бэк (по выбору
  пользователя); API останется независимым и переиспользуемым (моб. приложение в v3).
- **Python/FastAPI на бэке** — рабочий вариант, но разный язык с фронтом усложняет
  переиспользование типов; NestJS выигрывает по цельности стека.
- **MongoDB** — для финансов/заказов реляционная модель безопаснее; Postgres предпочтительнее.
