# Дизайн-система магазина

Стиль: **корпоративный, надёжный, международный**, в духе профессиональных рекламных
посадочных страниц Google (чистая сетка, много воздуха, ясная иерархия, конверсионные акценты).
Ниша товаров ещё не зафиксирована — дизайн-система строится так, чтобы быть **нейтральной к нише**
(токены и компоненты меняются один раз в конфиге, а не по всему коду), и одинаково хорошо работать
на английском и других языках (см. раздел 8 в docs/FEATURES_AND_HIGHLIGHTS.md про i18n).

## 1. Принципы дизайна

1. **Доверие превыше эффектов.** Каждая страница должна отвечать на "можно ли этому магазину
   доверять деньги и данные": видимые гарантии, отзывы, понятные условия возврата, контакты,
   защищённая оплата — на видном месте, а не в футере мелким шрифтом.
2. **Ясность через пространство.** Много whitespace, ограниченное число акцентных цветов на экран,
   один явный основной CTA на каждом экране.
3. **Производительность — часть дизайна.** Никаких тяжёлых hero-видео без lazy-load, все анимации
   ≤200мс и уважают `prefers-reduced-motion`.
4. **Консистентность через токены.** Ни один цвет/отступ/радиус не задаётся "на глаз" в компоненте —
   только через дизайн-токены (ниже). Это то, что отличает "профессиональный" сайт от типового.
5. **Mobile-first, data-informed.** Верстаем от 375px вверх; для админки — приоритет на плотность
   данных и таблицы, для витрины — на изображения и конверсию.
6. **Доступность (a11y) — не опция.** WCAG 2.1 AA: контраст ≥4.5:1 для текста, фокус-стили на всех
   интерактивных элементах, навигация с клавиатуры, alt-тексты, aria-атрибуты в кастомных компонентах.

## 2. Дизайн-токены

### 2.1 Цвет

Базовая палитра вдохновлена корпоративной эстетикой Google (надёжность + международность), но это
**отправная точка** — при появлении фирменного бренда меняется только файл токенов, не компоненты.

| Роль | Токен | HEX | Использование |
|---|---|---|---|
| Primary | `color.primary.500` | `#1A73E8` | Основной CTA, ссылки, активные состояния |
| Primary hover | `color.primary.600` | `#1558B0` | Hover/active кнопок |
| Primary tint | `color.primary.50` | `#E8F0FE` | Фоны выделения, бейджи "выбрано" |
| Success | `color.success.500` | `#1E8E3E` | Заказ подтверждён, в наличии |
| Warning | `color.warning.500` | `#F9AB00` | Промо, "осталось мало" |
| Danger | `color.danger.500` | `#D93025` | Ошибки, распродажа, "нет в наличии" |
| Neutral 900 | `color.neutral.900` | `#202124` | Основной текст |
| Neutral 600 | `color.neutral.600` | `#5F6368` | Вторичный текст |
| Neutral 300 | `color.neutral.300` | `#DADCE0` | Границы, разделители |
| Neutral 50 | `color.neutral.50` | `#F8F9FA` | Фон страницы |
| Surface | `color.surface` | `#FFFFFF` | Карточки, модалки |

Тёмная тема — зеркальная шкала neutral (900↔50), Primary остаётся тем же для узнаваемости бренда.
Каждый семантический цвет имеет шкалу 50/100/…/900, чтобы хватало оттенков для hover/disabled/badge.

### 2.2 Типографика

- Шрифт: **Inter** (свободная лицензия, геометрия близка к корпоративному Google-стилю; альтернатива
  — Manrope). Моноширинный для админ-таблиц с числами: **JetBrains Mono** или `tabular-nums`.
- Модульная шкала (1.25 — major third), базовая единица 16px:

| Стиль | Размер / высота строки | Вес | Применение |
|---|---|---|---|
| Display | 48/56 | 700 | Hero-заголовки лендинга |
| H1 | 36/44 | 700 | Заголовок страницы |
| H2 | 28/36 | 600 | Секции |
| H3 | 22/30 | 600 | Подзаголовки, карточки |
| Body L | 18/28 | 400 | Вступительные абзацы |
| Body | 16/24 | 400 | Основной текст |
| Body S | 14/20 | 400 | Вторичный текст, подписи |
| Caption | 12/16 | 500 | Бейджи, лейблы форм |

### 2.3 Сетка и отступы

- Базовая единица отступов — 4px: `4, 8, 12, 16, 24, 32, 48, 64, 96, 128`.
- Контейнер контента: `max-width: 1280px` (wide: 1536px для дашбордов), боковые поля 16px (mobile) /
  32px (desktop).
- Сетка: 4 колонки (mobile), 8 (tablet), 12 (desktop), gutter 24px.
- Брейкпоинты: `sm=640 md=768 lg=1024 xl=1280 2xl=1536`.

### 2.4 Радиусы и тени

- Радиусы: `4` (инпуты), `8` (кнопки, карточки), `16` (модалки, крупные карточки), `full` (бейджи,
  аватары, pill-кнопки).
- Тени — 4 уровня elevation: `e0` (нет, плоские элементы), `e1` (карточка в покое, едва заметная),
  `e2` (карточка при hover / dropdown), `e3` (модалка/поповер), `e4` (toast/уведомление поверх всего).

### 2.5 Иконки и иллюстрации

- Иконки: outlined-стиль, сетка 24×24, толщина линии 1.5–2px (совместимо с Lucide/Material Symbols
  Outlined) — избегаем "мультяшных" filled-иконок ради корпоративного ощущения.
- Иллюстрации для пустых состояний (пустая корзина, "заказов пока нет") — минималистичные
  line-art в 1–2 акцентных цветах, не стоковые картинки.

## 3. Библиотека компонентов

Реализуется как переиспользуемые React-компоненты (`src/components/ui/*`) поверх Tailwind +
shadcn/ui как базового генератора, но с собственными токенами — не "коробочный" вид.

- **Buttons**: primary / secondary / outline / ghost / danger × размеры `sm/md/lg` × состояния
  `default/hover/active/disabled/loading` (спиннер внутри кнопки, не блокирующий layout).
- **Forms**: text input, textarea, select (кастомный, не нативный `<select>`), checkbox, radio,
  switch, qty-stepper, date picker (для доставки), inline-валидация с понятными сообщениями об
  ошибках под полем.
- **Cards**: product card (изображение 1:1 или 4:5, бейдж скидки, рейтинг, цена/старая цена, быстрое
  добавление в корзину по hover), category tile, testimonial card, feature/benefit card (иконка +
  заголовок + текст, используется в трастовых блоках).
- **Navigation**: sticky header (лого, поиск, аккаунт, корзина со счётчиком), мега-меню категорий на
  desktop, off-canvas drawer-меню на mobile, хлебные крошки, пагинация + вариант "показать ещё".
  Компоненты: badge/tag (New, Sale −20%, Out of stock, "Бестселлер"), rating stars (частично
  заполненные), progress stepper (шаги оформления заказа: Корзина → Доставка → Оплата → Готово).
- **Overlays**: модалка (quick view товара), боковая панель корзины (slide-in справа, не уводит со
  страницы), toast-уведомления (успех/ошибка), skeleton-загрузчики (вместо спиннеров — контуры
  карточек/строк, чтобы не "прыгал" layout, CLS ≈ 0).
- **Data (админка)**: таблица с сортировкой/фильтрами/пагинацией на сервере, вкладки (Tabs),
  аккордеон (FAQ, характеристики товара), графики (line/bar) для дашборда продаж, кнопки
  bulk-действий (выбрать несколько строк → изменить статус).

## 4. Спецификации ключевых экранов

**Главная страница** — hero-баннер с чётким value proposition и одной кнопкой CTA (в духе
рекламного лендинга: заголовок-обещание, подзаголовок, кнопка, без визуального шума) → полоса
доверия (платёжные системы, "доставка по 190 странам", SSL-бейдж) → сетка популярных категорий →
карусель бестселлеров/новинок → блок УТП (3–4 карточки: гарантия, быстрая доставка, поддержка 24/7,
возврат 30 дней) → отзывы клиентов с фото/рейтингом → подписка на рассылку → футер-карта сайта.

**Каталог/листинг** — слева фильтры (цена-слайдер, категории, атрибуты, наличие; на mobile —
выезжающая панель), сверху панель сортировки и переключатель grid/list, чипы применённых фильтров
со сбросом, сетка карточек товара, пагинация или infinite scroll с "Load more".

**Карточка товара** — галерея изображений слева (зум по hover/pinch, миниатюры), справа sticky-блок
покупки (цена, наличие, выбор варианта — размер/цвет, qty, кнопка "В корзину" + "Купить в 1 клик"),
ниже — табы Описание / Характеристики / Отзывы, блок "Похожие товары" и "Вы недавно смотрели".

**Корзина** — слайд-панель для быстрых правок (qty, удаление) + отдельная полная страница для
финального просмотра: список позиций, поле промокода, sticky-блок итогов (подытог, скидка,
доставка, итого), явный CTA "Оформить заказ".

**Оформление заказа** — многошаговая форма с прогресс-индикатором (Контакты → Доставка → Оплата →
Подтверждение), возможность оформить без регистрации (guest checkout), автозаполнение адреса,
явное отображение способов оплаты с иконками провайдеров, страница успеха с номером заказа и
таймлайном статуса.

**Личный кабинет** — боковая навигация (Заказы, Адреса, Избранное, Настройки, Выход), таблица
заказов со статусами (badge-цвета соответствуют семантическим токенам), детальная страница заказа
с таймлайном доставки.

**Админ-панель** — отдельная тема (более плотная, табличная), боковой сайдбар разделов (Товары,
Заказы, Категории, Промокоды, Клиенты, Аналитика, Настройки), дашборд с ключевыми метриками
(выручка, заказы за сегодня, конверсия, топ-товары) в виде карточек + графиков, редактор товара с
drag&drop загрузкой изображений и предпросмотром карточки в реальном времени.

**Трастовые/статические страницы** — О компании, Контакты, FAQ, Доставка и возврат, Публичная
оферта/условия — обязательны для международного B2C и повышают конверсию и доверие платёжных
систем при подключении эквайринга.

## 5. Моушн и интерактивность

- Hover/press на кнопках и карточках — 150–200мс ease-out, лёгкий scale (1.00→1.02) или подъём тени.
- "Полёт" товара в иконку корзины при добавлении — короткая, приятная обратная связь без задержки
  основного действия.
- Скролл-reveal для маркетинговых секций главной страницы (fade+translateY 16px, однократно).
- Переходы между страницами каталога — быстрый fade, без полноэкранных анимаций, чтобы не тормозить
  восприятие скорости.
- Все анимации отключаются/упрощаются при `prefers-reduced-motion: reduce`.

## 6. Контент и тон

- Копирайтинг: короткие, выгодо-ориентированные формулировки (что получает покупатель, а не что
  умеет магазин), единый глоссарий терминов (не путать "Корзина/Заказ/Оформление" в разных местах).
- Весь UI-текст выносится в i18n-ключи с самого начала (даже если сейчас один язык) — это дешевле,
  чем переписывать позже под мультиязычность для международного рынка.
- Фотография/изображения товаров — единый стиль (белый/нейтральный фон для карточек каталога,
  контекстные фото в галерее товара), единое соотношение сторон в сетке — иначе "плывёт" сетка и
  ощущение непрофессионально.

## 7. Что закрепить в коде на Фазе 0/6 (см. DEVELOPMENT_PLAN.md)

- Файл токенов (`tailwind.config.ts` theme extend + CSS-переменные) — единственный источник правды
  по цвету/типографике/радиусам/теням.
- Storybook (или аналог) для витрины компонентов — так дизайн проверяется в изоляции до сборки
  страниц, особенно важно при разработке "по кусочкам" в разных сессиях.
- Lighthouse/PageSpeed бюджет: LCP < 2.5с, CLS < 0.1, INP < 200мс — фиксируем как критерий приёмки
  дизайна, а не только "красиво выглядит".
