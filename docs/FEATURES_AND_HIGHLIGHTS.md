# Функции и фишки магазина (детальный каталог)

Рынок — международный, бренд — корпоративный/надёжный. Ниже — полный разбор функциональности по
доменам: от MVP до продвинутых "фишек". Каждый пункт — с описанием, зачем он нужен и как устроен
технически, чтобы будущая сессия могла взять конкретный пункт и реализовать его без домышливания.
Приоритет: **MVP** (без этого магазин не работает) → **Growth** (повышает конверсию/удержание) →
**Advanced** (дифференциаторы, делают магазин "мощным").

## 1. Каталог и мерчандайзинг

**MVP**
- Категории деревом (неограниченная вложенность), товар может состоять в нескольких категориях.
- Товар с вариантами (размер/цвет/комплектация) — у каждого варианта своя цена, SKU, остаток,
  изображения.
- Гибкие атрибуты через JSON-поле (`Product.attributes`), чтобы не переделывать схему БД под
  каждую новую нишу/категорию.
- Управление остатками (inventory), автоскрытие/бейдж "нет в наличии".
- SEO: человекочитаемые slug-URL, meta title/description на уровне товара и категории,
  автогенерация `sitemap.xml`, JSON-LD `Product`/`Offer`/`BreadcrumbList` для расширенных
  сниппетов в поиске.

**Growth**
- Быстрый просмотр товара (quick view) без перехода на страницу.
- "С этим покупают" / "Похожие товары" — на старте: по категории+ценовому диапазону, позже —
  на основе истории заказов.
- Сравнение товаров (до 4 карточек side-by-side по атрибутам).
- Галерея с зумом и видео на карточке товара.

**Advanced**
- Персонализированная выдача каталога (сортировка по релевантности на основе поведения
  пользователя — просмотры/покупки).
- Динамическое ценообразование по сегментам (опт/розница, регион, валюта).
- Управление вариантами через матрицу (быстрое массовое редактирование цен/остатков в админке).

## 2. Поиск и обнаружение товара

**MVP**
- Полнотекстовый поиск на PostgreSQL (`tsvector`) по названию/описанию/атрибутам с релевантностью.
- Автодополнение (подсказки при вводе, top-5 совпадений).
- Фильтры: цена (слайдер), категория, атрибуты, наличие; сортировка (цена, новизна, популярность).

**Growth**
- Поиск с опечатками/синонимами и фасетным подсчётом ("Категория (23)") — переход на
  Meilisearch/Typesense, когда каталог вырастет (Postgres FTS не масштабируется на такое удобно).
- История поиска пользователя, "популярные запросы".

**Advanced**
- Поиск по изображению (загрузить фото → похожие товары) — на базе image-embedding модели,
  фича для дифференциации от типовых магазинов.
- Голосовой поиск (Web Speech API) для мобильной версии.

## 3. Корзина и оформление заказа

**MVP**
- Корзина для гостя (по cookie/сессии) и авторизованного пользователя, слияние корзин при логине.
- Гостевой чекаут (без обязательной регистрации — критично для международной конверсии).
- Расчёт доставки (фиксированные зоны/тарифы на старте), расчёт налогов/НДС по стране покупателя.
- Промокоды (фиксированная сумма / процент / бесплатная доставка), с условиями (мин. сумма, даты
  действия, лимит использований).
- Письмо-подтверждение заказа (очередь на отправку, чтобы не блокировать оформление).

**Growth**
- Сохранённые адреса и способы оплаты для повторных покупок.
- "Купить в 1 клик" с сохранённой картой/адресом по умолчанию.
- Прогресс-бар "до бесплатной доставки осталось N ₽/$" в корзине — доказанно повышает средний чек.
- Восстановление брошенной корзины (email через N часов, если оформление не завершено).

**Advanced**
- Рассрочка/BNPL интеграции (для международного рынка — Klarna/Afterpay-подобные провайдеры).
- Мультивалютная корзина с реальным курсом на момент заказа.
- Подписка на регулярные покупки (subscribe & save) для расходных товаров.

## 4. Оплата и ценообразование

**MVP**
- Абстракция `PaymentProvider` (единый интерфейс: создать платёж, обработать вебхук, вернуть
  средства) — конкретные провайдеры подключаются как плагины, не меняя ядро.
- Минимум два провайдера для международного рынка: один карточный эквайринг (Stripe) + локальный
  (ЮKassa/др. по региону) — выбираются по стране покупателя.
- Идемпотентная обработка вебхуков (защита от повторной обработки одного и того же события),
  сверка суммы заказа с суммой платежа перед подтверждением.
- Страницы результата: успех/ошибка/ожидание оплаты, статус заказа синхронизирован с платежом.

**Growth**
- Частичные возвраты из админки.
- Хранение платёжных методов пользователя (токенизация на стороне провайдера, не в своей БД).
- Автоматические счета/чеки (PDF) на email.

**Advanced**
- Мультивалютность с автоопределением по геолокации/выбору пользователя.
- Фрод-мониторинг (базовые эвристики: скорость заказов с одного IP/карты, лимиты).

## 5. Аккаунты и персонализация

**MVP**
- Регистрация email+пароль (хэш с bcrypt/argon2), восстановление пароля по email.
- OAuth (Google) — быстрый вход, важно для международной аудитории.
- Личный кабинет: история заказов со статусами, адреса, данные профиля.

**Growth**
- Список избранного (wishlist), доступный и гостю (localStorage) с переносом при регистрации.
- Уведомление "товар снова в наличии" (подписка на конкретный SKU).
- Программа лояльности / бонусные баллы за покупки, сгораемые/накопительные.

**Advanced**
- Реферальная программа (пригласи друга — скидка обоим).
- Сегментация пользователей для персональных промо (VIP, новые, "спящие" клиенты) — основа для
  будущих email-кампаний.

## 6. Админ-панель и операции

**MVP**
- CRUD товаров/категорий/вариантов с загрузкой изображений (drag&drop, авто-генерация превью).
- Управление заказами: смена статуса (новый → в обработке → отправлен → доставлен → отменён),
  печать накладной/инвойса.
- Управление промокодами.
- Ролевой доступ (RBAC): admin / manager / support — не у всех сотрудников должны быть все права.
- Журнал действий (audit log) — кто и когда изменил цену/статус заказа, обязательное требование
  для "мощной" и подотчётной админки.

**Growth**
- Дашборд метрик: выручка по периодам, конверсия воронки (просмотр→корзина→заказ→оплата), топ
  товаров, средний чек.
- Массовые операции (bulk edit цен/остатков, экспорт/импорт каталога через CSV).
- Управление контентом главной страницы (баннеры, подборки) без деплоя кода — мини-CMS на
  JSON-конфиге в БД.

**Advanced**
- Управление мультискладами и резервированием остатков.
- Прогноз спроса/остатков (простая эвристика на истории продаж → сигнал "пора закупать").
- Встроенная тикет-система поддержки клиентов (ответ на вопросы по заказу прямо из админки).

## 7. Маркетинг и рост ("фишки", отличающие от типового магазина)

- **A/B-тестирование лендингов** — вариативность hero-баннера/CTA с замером конверсии
  (в духе того, как оптимизируют рекламные посадочные страницы Google Ads).
- **UTM-трекинг и атрибуция** заказов к рекламным кампаниям — обязательно, раз стиль/подход
  ориентирован на performance-маркетинг.
- **Динамические скидки** (flash-sale таймеры, "количество ограничено" индикаторы — с честными,
  не поддельными данными об остатках).
- **Email/push-маркетинг**: welcome-серия, брошенная корзина, пост-покупочный follow-up,
  реактивация "спящих" клиентов — через очередь (BullMQ) + шаблоны писем.
- **Реферальная и партнёрская программа** (уникальные ссылки, начисление комиссии).
- **Отзывы с фото/видео** и верификацией "подтверждённая покупка" — сильно влияет на доверие
  международных покупателей.
- **Виджет доверия**: живые счётчики ("127 человек купили это на этой неделе"), значки платёжных
  систем, гарантии возврата — стандарт для performance-ориентированных лендингов.
- **Landing page builder** для рекламных кампаний — облегчённые страницы под конкретный товар/акцию
  с собственным URL для трафика из Google/Meta Ads, отдельные от основного каталога, чтобы быстро
  тестировать креативы.

## 8. Интернационализация (важно — рынок международный)

- **i18n с первой версии**: все строки интерфейса через ключи (`next-intl`/`i18next`), даже если
  сейчас 1 язык — дешевле, чем переписывать позже.
- **Мультивалютность**: цены хранятся в базовой валюте + таблица курсов, отображение в валюте
  пользователя.
- **Локализация форматов**: даты, номера телефонов, адресные формы (разные страны — разная структура
  адреса: индекс/штат/провинция и т.п.).
- **Региональные способы оплаты и налоги** (VAT/GST по стране), различное регулирование чеков.
- **CDN и edge-кэширование** статики для приемлемой скорости из разных регионов при хостинге на
  одном сервере.

## 9. Аналитика и данные

- Событийная аналитика воронки (просмотр товара → добавление в корзину → чекаут → оплата) —
  собственная (privacy-friendly) + интеграция с GA4/аналогом для рекламных кампаний.
- Server-side отслеживание конверсий для рекламных кабинетов (Google Ads/Meta CAPI) — точнее,
  чем клиентский пиксель, и меньше теряется из-за блокировщиков рекламы/ITP — логичное дополнение
  к общей эстетике и цели "performance-маркетинг стиль".
- Когортный анализ удержания клиентов (для программы лояльности/email-кампаний).

## 10. Технические "фишки" платформы

- **PWA**: офлайн-доступ к каталогу (кэш последних просмотренных), "Установить на телефон",
  push-уведомления о статусе заказа.
- **ISR/on-demand revalidation**: карточки товара обновляются мгновенно при изменении в админке,
  без полного ребилда сайта.
- **Оптимизация изображений**: автоматическая генерация нескольких размеров/форматов (AVIF/WebP)
  при загрузке в админке через sharp, `srcset` на фронте.
- **Feature flags**: включение/выключение фич (новый чекаут, новый дизайн карточки) без деплоя —
  критично при разработке "по кусочкам" в разных сессиях, чтобы незавершённые фичи не ломали прод.
- **API-first ядро**: все операции доступны через внутренний REST/GraphQL слой — облегчает будущее
  мобильное приложение без переписывания бэкенда.

## 12. Открытые решения, влияющие на объём работы

- Точная ниша товаров (когда определится) может добавить специфичные атрибуты/фичи (например,
  конфигуратор для техники, размерные сетки для одежды) — тогда в раздел 1 добавляется подраздел
  под нишу.
- Список языков/валют для запуска (сколько локалей поддерживаем в MVP, а не "когда-нибудь").
- Какие рекламные каналы (Google Ads/Meta/др.) нужно интегрировать в первую очередь для раздела 9 —
  от этого зависит, какие конверсионные API подключаем раньше.
